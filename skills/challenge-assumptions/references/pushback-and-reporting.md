# Pushback And Reporting

Read this reference when user pushback changes the evidence, or when a Full investigation needs a durable result.

## Process User Pushback

Treat a new, specific user-observed fact as evidence. Treat the user's causal explanation as a hypothesis.

1. Restate the observed fact separately from the explanation.
2. Lower confidence only in premises that conflict with the fact; preserve unrelated verified evidence.
3. Check whether the disputed premise relied on authoritative, current, scope-equivalent evidence.
4. If the fact opens a material unresolved branch, choose the cheapest safe, authorized differential whose decision value justifies its cost, delay, disruption, and residual risk.
5. Run that probe before repeating the earlier conclusion. If no justified probe exists, state the unresolved branch and constrain the conclusion and next action to the remaining evidence.
6. Correct affected claims individually before giving new steering.

An implausible theory without a new observation does not become a mandatory branch. Subject every proposed theory to the same materiality and decision-value threshold.

## Report A Micro Investigation

State:

- The conclusion and confidence.
- The decisive probe and result.
- The decision delta.
- Any material residual uncertainty.

## Report A Full Investigation

Use a compact ledger when it helps the user act:

```text
Verified observations:
- <authoritative, current, scope-equivalent facts>

Causal inference:
- <explanation, supporting observations, and confidence>

Rejected:
- <hypotheses validly falsified>

Still possible:
- <material unresolved hypotheses>

Next:
- <narrow fix, validation, or blocker>

Decision delta:
- Before probe: <likely action or conclusion>
- After probe: <changed or confirmed action or conclusion>
```

Do not expose internal ceremony when it makes a simple answer harder to use.

## Final Check

- Distinguish verified observations from causal inference.
- Label stale, proxy, partial, or differently scoped evidence.
- Do not count repeated views of one source or repeated failures on one path as independent evidence.
- Do not stop at "metadata is visible but action is blocked" when a safe, authorized, worthwhile probe can locate the boundary; otherwise state that the boundary is unverified and name the blocker.
- Do not broaden a remedy beyond what authoritative evidence supports.
- Preserve unresolved material branches when testing is unavailable or unjustified.
